🛡 SOBRE O ERRO "Uncaught tutor (linha 3433)"
  Esse erro NÃO existe no código da v9.8 (auditado). Ele veio de
  uma versão ANTIGA presa no cache do celular pelo service worker
  antigo. O sw.js novo resolve de vez: agora o app é
  "network-first" — sempre que houver internet ele busca a versão
  mais nova do site, e o cache só responde offline. Além disso,
  ao ativar, ele APAGA todos os caches antigos.
  Bônus: durante a investigação encontrei e corrigi um travamento
  parecido que aconteceria ao salvar paciente quando ainda não há
  NENHUM tutor cadastrado — blindado na v9.8.

🔑 GERADOR DE CÓDIGOS (gerador.html) — ANTIFRAUDE v9.9
  Códigos PRO agora são VINCULADOS AO APARELHO do cliente:
  1. O cliente abre a tela do Pro no app e copia o ID
     (ex: AP-K7M2) — o botão do WhatsApp já envia o ID junto
  2. Você digita esse ID no gerador e gera o código
  3. O código SÓ ativa naquele aparelho — em outro celular
     é recusado. Compartilhar código não funciona mais.
  • O ID viaja dentro do backup: se o cliente trocar de
    celular e RESTAURAR o backup, a licença dele continua.
  • Convites BETA seguem universais (acesso grátis).
  ⚠ Códigos Pro antigos (universais) deixaram de valer —
    quem já está com Pro ATIVADO não é afetado; para novas
    ativações, gere no formato novo.
  Algoritmos idênticos aos do app v9.9.
